# Documentation Factory — Autonomous Codebase Intelligence

An autonomous, multi-agent pipeline dedicated to deeply analyzing enterprise codebases and automatically generating production-grade, "Docs-as-Code" web portals (MkDocs). It operates deterministically, guaranteeing zero hallucinated links and 95%+ coverage accuracy.

## What It Does
Point it at a source codebase (Terraform, Kubernetes, Apps, Pipelines). It autonomously:
1. **Discovers** the full dependency graph.
2. **Plans** a safe, memory-efficient documentation strategy (Wave-Based Processing).
3. **Analyzes** code, extracting architecture, variables, pipelines, and deducing **Architecture Decision Records (ADRs)**.
4. **Audits** the generated specs deterministically against the source code for 100% coverage.
5. **Compiles** the raw markdown into a fully navigable MkDocs static site.
6. **Validates** all markdown formatting and guarantees zero dead internal links.
7. **Publishes** the final compiled website to a Git feature branch.

## Agents

| # | Agent | Role | Invoked |
|:-:|-------|------|---------|
| 1 | **discovery-scanner** | Builds a deterministic census of the codebase using the `dep-graph-builder` python skill. | Pre-Wave |
| 2 | **doc-planner** | Batches files into isolated Waves to prevent LLM context-window exhaustion. | Pre-Wave |
| 3 | **spec-analyst** | Generates deep specs for IaC/App logic and deduces formal **ADRs**. | Per Wave |
| 4 | **flow-tracer** | Maps out CI/CD workflows and data control flows. | Per Wave |
| 5 | **variable-extractor** | Extracts inputs/outputs to build the `global-data-dictionary.json` (shared with Migration Factory). | Per Wave |
| 6 | **doc-reviewer** | Reconciles the LLM documentation against the deterministic filesystem census to guarantee coverage. | Per Wave |
| 7 | **doc-surgical-fix** | Surgically patches missing fields in the markdown using `git diff`. | On Gate Failure |
| 8 | **topology-mapper** | Generates Mermaid.js architectural diagrams. | Post-Wave |
| 9 | **doc-assembler** | Stitches outputs, generates the `mkdocs.yml` navigation tree. | Post-Wave |
| 10 | **site-builder** | Runs `markdownlint`, `lychee` (Dead-link check), and `mkdocs build --strict`. | Post-Wave |
| 11 | **shared-memory-writer**| Extracts global jargon to Categorized Knowledge rules (prevents future hallucinations). | Post-Wave |
| 12 | **doc-git-publisher** | Commits the final static site and pushes to a Git feature branch. | Post-Wave |

## Deterministic Python Skills

The Documentation Factory relies on "hard" Python validation tools rather than subjective LLM reviews to guarantee enterprise quality.

### 1. `coverage-auditor`
Replaces subjective LLM reviews. It parses the Markdown generated by the `spec-analyst`, extracts traceability tags (e.g., `files_covered: [main.tf]`), and runs a mathematical reconciliation against the `discovery-scanner`'s baseline census. If coverage is `< 95%`, it hard-fails the gate.

### 2. `mermaid-linter`
Mermaid syntax is notoriously difficult for LLMs to generate perfectly. This skill intercepts the output of the `topology-mapper` and uses the official `mmdc` (Mermaid CLI) to validate the graph syntax. If broken, it passes the exact syntax error back to the surgical-fix agent.

### 3. `dep-graph-builder` (Shared)
Instead of relying on the LLM to guess the repository structure, this tool generates a deterministic AST/Regex map of the repository, guaranteeing zero missing files in the inventory.

## DevSecOps & Docs-as-Code

This factory strictly adheres to DevOps principles:
- **Zero Data Loss:** Context is maintained via *Categorized Knowledge Routing*, meaning lessons learned are permanently stored in isolated files (e.g., `knowledge/auth.md`) rather than bloated into a single file.
- **Zero Dead Links:** The `site-builder` utilizes deterministic link auditing (`lychee`) to guarantee that every single LLM-generated cross-reference resolves to an actual file.
- **Automated Publishing:** Documentation doesn't sit on a local drive. The `doc-git-publisher` automates branching, committing, and pushing.
